## 1. 데이터 구조 이해

### 1-1. 엑셀 파일 시트 구성 (매달 동일 형식 유지)

| 시트명 패턴 | 내용 | 자동 인식 방법 |
|---|---|---|
| `최훈서YYMMDD` | 훈서 거래내역 (1년치) | 시트명에 "자산" 없음 → 거래 시트 |
| `훈서자산` | 훈서 재무현황 스냅샷 | 시트명에 "자산" 포함 → 자산 시트 |
| `윤시온YYMMDD` | 시온 거래내역 (1년치) | 시트명에 "자산" 없음 → 거래 시트 |
| `시온자산` | 시온 재무현황 스냅샷 | 시트명에 "자산" 포함 → 자산 시트 |

### 1-2. 거래내역 시트 컬럼 (10개 고정)
```
날짜 | 시간 | 타입 | 대분류 | 소분류 | 내용 | 금액 | 화폐 | 결제수단 | 메모
```
- **타입**: `수입` / `지출` / `이체`
- **금액**: 지출은 음수(-), 수입은 양수(+)
- **이체**는 저축/투자/보험/카드대금 포함 — 분석 시 수입/지출과 분리해서 처리

### 1-3. 자산 시트 구조 (섹션 번호로 파싱)
```
1.고객정보   → 이름, 성별, 나이, 신용점수(KCB)
2.현금흐름현황 → 월별 수입/지출 집계 (row 10이 헤더, col4~16이 월)
3.재무현황   → 자산 목록 + 부채 목록
              총자산 row: col4=총자산금액, col8=총부채금액
              순자산 row: 다음 row col1 = 순자산 금액
4.보험현황   → 보험사, 상품명, 계약상태, 납입금
5.투자현황   → 투자상품종류=주식, col2=증권사, col3=종목명,
              col5=투자원금, col6=평가금액, col7=수익률
6.대출현황   → 대출종류=은행 대출, col3=상품명,
              col6=대출잔액, col7=금리
```

if row[1] == '총자산':

if row[1] == '순자산':
    net_assets = float(rows[i+1][1])   # ← 다음 행에서 가져옴

### 3-2. 컬럼 구조
```csv
keyword,category,desc_label,notes
```
- **keyword**: 거래내역 `내용` 컬럼에서 매칭할 검색어 (대소문자 무시)
- **category**: 변경할 카테고리명
- **desc_label**: 표시할 깔끔한 이름 (빈칸이면 원본 유지)
- **notes**: 메모 (처리에 영향 없음)

### 3-3. 현재 핵심 분류 규칙
| keyword | 변경 전 | 변경 후 | 이유 |
|---|---|---|---|
| `85690900042583` | 의료/건강 | 금융 | **아이엠삼송 전세이자** — 매월 21일 정기 출금 |
| `업비트 입금` | 온라인쇼핑 | 투자 | 가상자산 투자 |
| `대출결산이자` | 금융 | 금융 | 하나은행 마이너스 통장 이자 |
| `전기차충전(정기)_스마트로` | 자동차 | 자동차 | Tesla 정기 충전 (유지) |
| `메리츠통합004` | 금융 | 보험 | 메리츠화재 통합 보험료 |
| `ANTHROPIC` | 온라인쇼핑 | 구독 | Claude AI 구독 |

### 3-4. 적용 코드 패턴
```python
def apply_classification(desc, orig_cat, rules):
    for rule in rules:
        if rule['keyword'].lower() in str(desc).lower():
            return rule['category'], rule['desc_label'] or desc
    return orig_cat, desc
```

### 7-3. 대시보드 기능 목록
| 기능 | 설명 |
|---|---|

### 7-4. db/ CSV 스키마

**transactions.csv** (거래내역):
```
id, person, date, time, type, category, subcategory, desc, amount, currency, method, memo
```
- `id`: `h_0`, `w_0` 형식. person별 단조증가. 재사용 절대 금지
- `person`: `h`(훈서) / `w`(시온)
- `type`: `수입` / `지출` / `이체`
- `amount`: 지출 음수(-), 수입 양수(+)

**assets.csv** (자산 스냅샷):
```
snapshot_date, person, total_assets, total_liabilities, net_assets, credit_score,
dc_pension, irp_pension, investments_json, loans_json
```
- `snapshot_date`: `YYYY-MM` 형식. 같은 달 같은 person이면 덮어쓰기
- `investments_json`, `loans_json`: JSON 직렬화 문자열

**overrides.csv** (편집 내용):
```
id, cat, desc, deleted, updated_at
```
- 대시보드 수정/삭제 시 serve.py가 자동 관리. Claude가 건드리지 않음

### 7-5. serve.py API 엔드포인트

| 메서드 | 경로 | 설명 |
|---|---|---|
| GET | `/` | dashboard.html 서빙 |
| GET | `/api/data` | 전체 데이터 + 집계 JSON 반환 |
| POST | `/api/override` | `{"id":"h_0","cat":"금융","desc":"이름"}` |
| POST | `/api/delete` | `{"id":"h_0"}` |
| POST | `/api/bulk-override` | `{"ids":[...],"cat":"금융","desc":null}` |

### 7-6. 금액 포맷 규칙
```javascript
function fmt(n) {
  if (n >= 100000000) return (n/100000000).toFixed(2) + '억';   // 1.35억
  if (n >= 10000000)  return (n/10000000).toFixed(2) + '천만'; // 1.27천만
  if (n >= 10000)     return Math.round(n/10000) + '만';        // 127만
  return n.toLocaleString();                                      // 1,270
}
// 수익률: (v>=0?'+':'') + v.toFixed(2) + '%'  → +103.71%
```

### 9-1. 이 가계의 특성 요약
- **맞벌이 부부**, 훈서(메리츠화재) + 시온(큐브리드), 결혼 준비 중 추정
- **훈서 주도 지출**: 지출의 87%가 훈서. 자동차(Tesla), 골프, 여행 등 라이프스타일 지출 많음
- **시온 안정형**: 부채 없음, 신용점수 만점, 꾸준한 저축 패턴
- **수입보다 지출이 많은 구조**: 성과급이 없으면 연간 적자 → 성과급 관리가 핵심
- **투자 성향 (훈서)**: 레버리지 ETF, 코인 ETF 등 공격적. 메리츠금융지주만 큰 수익
- **투자 성향 (시온)**: 빅테크 분산 (MSFT, GOOGL, AAPL, NVDA). 안정적이나 현재 마이너스

### 9-2. 매달 주목해야 할 지표
1. **순자산 변화**: 전월 대비 증가/감소 — 투자 평가 변동 + 저축 + 지출 합산
2. **지출 이상치**: 평균 대비 2배 이상 지출 항목 강조
3. **투자 수익률 변화**: 종목별 월별 추이 (자산 시트 새로 올라올 때마다)
4. **시온 급여 추이**: 큐브리드 급여가 꾸준히 오르고 있음 (348만→714만 1년 사이)
5. **업비트 잔고**: 코인 투자 현황 (출금이 있을 때 수익/손실 추정)
